Instructions

The main goal is for you to make a web application that will summarize and display data found here: https://rawgit.com/jwalton922/frontendTestProject/master/FrontEndTestData.json Your application should be viewable by clicking the 'Your Page' link above.

Most of the data has five attributes: industry, state, number of employees, revenue, and date founded. Using this application, a user could answer questions like: "How many consulting companies in NY have more than 10 employees?" or "Have there been any manufacturing companies founded in California in the last five years?"

Requirements

  • Make all of your code changes off a branch. Create a pull request to submit your code when done.
  • Your app has to retrieve the data through an HTTP request in an Angular service.
  • After the data is loaded, display the values for each attribute in it's own table along with the total times that value was found
  • Transform date founded into age in years
  • Transform employees, revenue, and age into ranges so that there are not so many unique values. For instance, employees could be reported in 1-5, 5-10 10-25, 25-50,50+. Use at least three ranges for each attribute.
  • When a value in a table is clicked on, only show data (across all tables) that has that value. For instance, if a user clicks on "Manufacturing" in the Industry table, only show state, employee range, revenue range, and age range where the industry is "Manufacturing".
  • Provide some styling to make the content easy to look at. You won't be judged by design ability, but the content needs to look clean.
  • When your project is evaluated, it will be cloned fresh, your branch will be checked out, 'bower install' will be run, then 'grunt serve'. Make sure these steps work for you before submitting.
  • Please don't spend much more than a couple of hours on this. If you couldn't finish, let me know what problems you encountered and what your next steps were going to be.

Bonus Requirements

  • Use Bootstrap for styling - due to a bug, it's not included in the app. Figure out a way to include it and use it for your application view.
  • Allow each table to also be viewed as a Pie Chart. Use any charting library you like.
